# FactionGen — Social Entity & Faction Generator
**Repository:** `gi7b-factiongen` (standalone open-source repo)
**Status:** Specification v1.0
**License:** MIT

---

## Overview

FactionGen is a standalone faction generation library. It takes **world-level inputs from MWG** — Population, Wealth, Development, Power Structure, Source of Power, and Tech Level — and produces a coherent set of **Entities** (factions, organizations, institutions) that plausibly emerge from those conditions.

Entities name themselves using `gi7b-namegen` (personal names for leaders and members) and `gi7b-placegen` (place-based names and territorial identifiers), biased toward the cultural values generated by the D66×D6 Culture Table.

FactionGen ships as a standalone PWA with web tools for browsing, editing, and packaging generated faction sets.

---

## Design Philosophy

Factions are treated as **organisms**, not static labels. They have finite resources, capabilities that can grow or decay, and relationships that emerge from competing interests — not from a relationship matrix rolled in a vacuum.

The MWG world stats are causal inputs, not flavour. A high-Wealth / low-Development world (Dutch Disease conditions) produces a different faction landscape than a high-Development / low-Wealth one. The generator models this causally.

**Entity** is the canonical term. Factions, corporations, governments, cults, guilds, gangs, and noble houses are all Entities. The Entity stat block uses the same six-attribute structure as CE/Traveller characters, adapted to organisational scale.

## MWG Input Interface

FactionGen consumes the following MWG world outputs:

| MWG Field | Type | Notes |
|---|---|---|
| `population` | integer | Actual population (not Bp) |
| `wealth` | `Average \| Better-off \| Prosperous \| Affluent` | From MWG Wealth Table |
| `development` | `UnderDeveloped … Very Developed` | From MWG Development Table (HDI proxy) |
| `powerStructure` | `Anarchy \| Confederation \| Federation \| Unitary State` | From MWG Power Structure Table |
| `sourceOfPower` | `Aristocracy \| Ideocracy \| Kratocracy \| Democracy \| Meritocracy` | From MWG Source of Power Table |
| `mtl` | integer 1–16 | Mneme Tech Level (internal canonical value) |
| `cultureRolls` | `CultureEntry[]` | 1–N results from D66×D6 Culture Table |
| `lcWeights` | `Record<string, number>` | Campaign LC weight preset (from PlaceGen/NameGen) |

These inputs determine:
- How many Entities exist
- What types of Entities dominate
- How Entity attributes are distributed
- What ideological axes Entities compete along

## Economic Assumptions

Before distributing the population into factions, FactionGen computes world economic conditions from MWG inputs:

### Wealth–Development Decoupling

Wealth and Development are independent axes. Their combination determines the economic character of the world:

| | Low Development | High Development |
|---|---|---|
| **High Wealth** | Dutch Disease: resource extraction dominates; institutions weak; wealth concentrated in few Entities; high Kratocracy/Aristocracy pressure | Mature economy: broad institutional base; many mid-tier Entities; Democracy/Meritocracy more stable |
| **Low Wealth** | Scarcity: Entities compete fiercely for limited resources; fragmentation high; Anarchy more likely | Development without wealth: strong institutions, low material power; Ideocracy/Meritocracy dominant |

### Tech Level Effects

MTL affects Entity type distribution and capability ceilings:

| MTL Range | Effect on Factions |
|---|---|
| 1–5 | Pre-industrial: Entities are clans, noble houses, religious orders, city-states |
| 6–8 | Industrial: Entities include corporations, unions, political parties, military branches |
| 9–11 | Information age: Entities include media conglomerates, intelligence agencies, NGOs, megacorps |
| 12–14 | Interstellar: Entities include jump-capable trade consortia, colonial governments, mercenary fleets |
| 15–16 | Post-scarcity: Entities may be post-human collectives, AI-adjacent institutions, megastructure authorities |

## Entity Count and Distribution

### Number of Entities

Entity count is derived from Population and Power Structure:

```
baseCount = floor(log10(population)) + powerStructureModifier
```

| Power Structure | Modifier | Character |
|---|---|---|
| Unitary State | −2 | Few dominant Entities; state consolidates power |
| Federation | 0 | Balanced plurality |
| Confederation | +2 | Many competing Entities; loose coordination |
| Anarchy | +4 | Fragmented; many small Entities; no dominant structure |

`baseCount` is then modified by Development (higher development → more institutional diversity → +1 to +3) and Wealth (extreme wealth concentration → fewer but larger Entities; −1 to −2 for Affluent + UnderDeveloped).

### Entity Type Distribution

The Source of Power determines which Entity types are most likely to emerge:

| Source of Power | Dominant Entity Types | Secondary Types |
|---|---|---|
| Aristocracy | Noble houses, hereditary estates, clan networks | Religious orders, merchant guilds |
| Ideocracy | Political movements, religious institutions, cultural academies | NGOs, media organs |
| Kratocracy | Military commands, warlord factions, paramilitary groups | Criminal organisations, mercenary bands |
| Democracy | Political parties, elected councils, civic associations | Unions, advocacy groups |
| Meritocracy | Professional guilds, research institutions, licensing bodies | Corporations, civil service |

## Entity Stat Block

Every Entity has six attributes using the standard CE/Traveller 2–12 scale:

| Attribute | Abbreviation | Meaning |
|---|---|---|
| **Strength** | STR | Manpower efficiency and productivity; the output per member |
| **Dexterity** | DEX | Organisational coordination; speed of decision and response |
| **Endurance** | END | Raw bodycount; total membership and staying power |
| **Intelligence** | INT | Competence; quality of expertise within the organisation |
| **Education** | EDU | Institutional knowledge; ability to train, recover losses, and retain capability over time |
| **Social** | SOC | Social capital, reputation, and income relative to the broader society |

**Important:** Entity SOC is **not** the average SOC of its individual members. It is the organisation's standing in society as a whole. An Entity's SOC is typically equal to or higher than the highest-ranking individual within it — the organisation confers status on its members, and the SOC of the organisation can in turn raise the SOC of individuals who belong to it.

Individuals within an Entity are measured in the abstract by their personal SOC and their competence level (a function of EDU and INT at individual scale).

### Attribute Generation

Entity attributes are not rolled in a vacuum. They are constrained by world conditions:

**END** is proportional to the Entity's population share:

```
entityPopulation = totalPopulation × populationShareFraction
END = populationShareToScore(entityPopulation)
```

**STR** is bounded by MTL (higher TL enables higher productivity per member).

**INT** is bounded by Development (higher HDI → higher institutional competence ceiling).

**EDU** is bounded by Development and TL (institutional knowledge requires both).

**SOC** is influenced by Source of Power alignment. An Entity whose type aligns with the world's Source of Power gains a SOC bonus.

**DEX** is influenced by Power Structure. Entities in Anarchy worlds tend toward low DEX (poor coordination); Unitary State worlds may have one or two high-DEX Entities (the state itself) and many low-DEX peripheral ones.

## Population Distribution Into Entities

The total world population is divided among Entities. Distribution is not uniform — it follows a power-law curve modified by Power Structure:

```
populationShares = generatePowerLawShares(entityCount, powerStructureConcentration)
```

| Power Structure | Concentration | Character |
|---|---|---|
| Unitary State | High | One Entity may hold 60–80% of END-equivalent population |
| Federation | Medium | 2–3 dominant Entities; long tail of minor ones |
| Confederation | Low-Medium | Several mid-size Entities competing |
| Anarchy | Very Low | Many small Entities; no dominant one |

Wealth concentration (Dutch Disease conditions) adds a secondary distortion: a small number of resource-extraction Entities hold disproportionate SOC and STR relative to their END.

## Cultural Values → Entity Ideological Axes

Culture Table rolls (D66×D6) produce a set of **cultural values** for the world. These values determine the **ideological axes** along which Entities compete and self-identify.

### Mapping Culture Rolls to Axes

Each Culture Table result implies one or more ideological dimensions. FactionGen maps these to named axes:

| Axis | Poles | Triggered By (examples) |
|---|---|---|
| Collectivism ↔ Individualism | Live for all ↔ I am number one | Row 2-1 |
| Tradition ↔ Innovation | Hallowed traditions ↔ Tradition shmadition | Row 3-4 |
| Hierarchy ↔ Equality | Absolute authority ↔ No government | Row 4-6 |
| Pacifism ↔ Militarism | Beyond war ↔ Perpetual war | Row 5-2 |
| Spirituality ↔ Secularism | Theological certainty ↔ Atheology | Row 3-2 |
| Purity ↔ Pragmatism | Zealotry ↔ Pragmatic spirituality | Row 3-3 |
| Openness ↔ Isolation | Xenophilia ↔ Xenocide | Row 3-5 |

Only axes triggered by the world's actual Culture Table rolls are active for that world. Each active axis becomes a dimension along which Entities are positioned.

### Entity Positioning on Axes

For each active axis, each Entity is assigned a position (−3 to +3). Position is influenced by Entity type and Source of Power alignment. Two Entities may share similar axis positions (ideological alignment) or oppose each other (competition), creating the raw material for faction relationships.

## Entity Naming

Entities name themselves using `gi7b-namegen` and `gi7b-placegen`, biased toward their cultural values.

### Name Construction

An Entity name is constructed from:

1. **Root name** — drawn from the PlaceGen or NameGen word pool, biased toward categories that resonate with the Entity's ideological position and type.
2. **Descriptor** (optional) — an adjective or noun from the Culture Table's active value set.
3. **Structural term** — reflects governance type (e.g., "Order", "Assembly", "Directorate", "Brotherhood", "Compact", "Bureau").

**Examples by type:**

| Entity Type | Source of Power | Example Name |
|---|---|---|
| Military command | Kratocracy | Iron Compact of the Northern Reaches |
| Religious institution | Ideocracy | The Order of the Guiding Light |
| Political party | Democracy | The Progressive Assembly |
| Noble house | Aristocracy | House Carbinter |
| Research guild | Meritocracy | The Thornfield Institute |

The LC pair from the campaign's weight preset determines the linguistic flavour of all name components.

## Entity Relationships

Relationships between Entities emerge from their axis positions and type interactions — they are not rolled independently.

### Relationship Generation

For each pair of Entities:

1. Compute **axis agreement score** — the sum of signed distances across all active axes.
2. Map agreement score to a base relationship type:

| Score | Base Relationship |
|---|---|
| Close agreement | Ally / Client |
| Moderate agreement | Competitor (cooperative rivalry) |
| Neutral | Indifferent |
| Moderate disagreement | Rival |
| Strong disagreement | Enemy |

3. Apply **type interaction modifiers** — e.g., a Kratocracy-aligned military Entity always has tension with Democracy-aligned civic Entities regardless of axis agreement.

4. Apply **resource pressure modifiers** — Entities competing for the same population pool have additional rivalry pressure.

### Nuance Dial

A referee-controlled nuance setting modulates relationship complexity:

| Setting | Effect |
|---|---|
| Low | Binary: ally or enemy; no contradictions |
| Medium | Allows cooperative rivals and conditional alliances |
| High | Full complexity: contradictory relationships, conditional loyalties, shifting allegiances |

## TypeScript Library API

```typescript
import { FactionGen } from 'gi7b-factiongen'
import { PlaceGen } from 'gi7b-placegen'
import { NameGen } from 'gi7b-namegen'

const world = {
  population: 600_000_000,
  wealth: 'Prosperous',
  development: 'Developed',
  powerStructure: 'Federation',
  sourceOfPower: 'Democracy',
  mtl: 10,
  cultureRolls: [ /* D66×D6 results */ ],
  lcWeights: { 'en-gb': 4, 'la-roman': 2 }
}

const gen = new FactionGen({ world, seed: 99 })
const factions = gen.generate()

// factions: Entity[]
// Each Entity:
// {
//   name: 'The Thornfield Assembly',
//   type: 'political_party',
//   population: 42_000_000,
//   attributes: { STR: 7, DEX: 9, END: 8, INT: 10, EDU: 9, SOC: 11 },
//   axisPositions: { collectivism: +2, tradition: -1, hierarchy: -2 },
//   relationships: [ { entity: 'Iron Compact', type: 'Rival', nuanceFlags: ['resource_competition'] } ],
//   keyTerms: ['assembly', 'progressive', 'civic', 'thornfield'],
//   governance: 'Federation',
//   sourceOfPower: 'Democracy'
// }
```

## PWA — Web Tools

FactionGen ships as a PWA in the `pwa/` directory.

**Tools included:**

### World Input Form
- Enter MWG world stats directly or paste from MWG export
- Adjustable LC weight presets (links to PlaceGen/NameGen)
- Culture Table roller (D66×D6) with manual override

### Faction Browser
- Generated Entities displayed as cards with full stat blocks
- Filter by type, SOC, alignment
- Drill into any Entity for full attribute breakdown and relationship map

### Relationship Map
- Force-directed graph of Entity relationships
- Color-coded by relationship type (ally / rival / enemy / indifferent)
- Nuance dial control

### Entity Editor
- Edit any Entity's attributes, name, type, or axis positions inline
- Changes propagate to relationship map in real time

### Package Builder
- Export full faction set as JSON package
- Packages importable into MWG or any consuming project
- Optional: export as PDF faction reference sheet

## Integration with MWG

FactionGen is called after the **Inhabitants** generation step (Step 7 of the MWG waterfall), once Population, Wealth, Development, Power Structure, Source of Power, and TL are all known:

```typescript
// In MWG inhabitant generation (post Step 7)
import { FactionGen } from 'gi7b-factiongen'

const factionGen = new FactionGen({ world: mainWorldInhabitants, seed: systemSeed })
starSystem.factions = factionGen.generate()
```

Factions are stored on the `StarSystem` record in MWG's IndexedDB alongside world data.
